# Getting Started

Make sure you have Jsonic installed. The installation instructions cover your various options and environments in more detail.

Then load into your source code:

# use require if you're writing for Node.hjs:
const { Jsonic } = require('jsonic')
# use import if you're writing for a browser:
import { Jsonic } from 'jsonic'

# Parse some easy-going JSON!

The Jsonic variable is a function that takes in a string and returns an object:

let data = Jsonic('a:1')
console.log( JSON.stringify(data) ) // prints {"a":1}

You pass in your jsonic source as a string. The string is parsed and converted into a JavaScript variable.

# What does jsonic syntax give you?

Standard JSON syntax almost provides a good developer experience, but not quite. We need a few extra things:

# Trailing commas are ignored

Jsonic('[ 1, 2, ]') // === [1, 2]
Jsonic('{ "a":3, "b":4, }') // === {"a":3, "b":4}

Just like a modern JavaScript. This one you can't live without.

# Object keys don't need to to be quoted

Jsonic('{a:1}') // === {"a":1}

Now object keys are more readable.

# Single quote strings work

Jsonic('"a"') // === 'a'
Jsonic("'a'") // === 'a'

Just like normal JavaScript.

# Backtick multi-line strings work

// === {"a":1, "b":2}
Jsonic(`{
  a: 1,
  b: 2,
`)

Again, just like JavaScript.

# You get all the comments

// === {"a":1, "b":2}
Jsonic(`{
  a: 1,
  // A comment
  # Also a comment
  /*
   * Still a comment
   */
  b: 2,
`)

Now you can comment your configuration files properly.

# Actually, who needs commas when newlines will do

// === { "a": 1, "b": [ 2, 3 ] }
Jsonic(`{
  a: 1
  b: [
    2
    3
  ]
`)

Why should YAML have all the fun?

# And quoting every string is tedious, right?

// === {"a":"foo", "b":"bar"}
Jsonic('{a:foo, b:bar}')

When a value isn't a number, true, false, or null, it has to be a string.

# Don't worry about spaces inside text either

// === { "a": "foo bar" }
Jsonic('{a: foo bar }')

Surrounding space is trimmed as you would expect.

# Implicit objects and arrays at the top level

// === {"a":1, "b":2}
Jsonic('a:1,b:2')

// === ["a", "b"]
Jsonic('a,b')

This is very convenient for quickly defining small objects.

# All the JavaScript numbers and string escapes

// === [20.0, 20, 20, 20, 20]
Jsonic('20.0, 2e1, 0x14, 0o24, 0b10100')

// === ['a', 'a', 'a']
Jsonic('"\x61", "\u0061", "\u{000061}"')

Parity with JavaScript string handling is important to Jsonic.

# Block text with indent removal

// === 'ship!\ndish!\nball!'
Jsonic(`
  '''
  ship!
  dish!
  ball!
  '''
`)

A great idea from HJson (opens new window), so we had to have it!

# Deep property one liners

// === { "a": { "b": { "c": 1 } } }
Jsonic('a: b: c: 1')

Also a great idea from the Cue configuration language (opens new window).

# Merging of duplicate properties

// === {"a": {"b":1, "c":2, "d": 3}}
Jsonic(`
  a: {b: 1}
  a: {c: 2}
  a: d: 3
`)

This is super useful when writing system specifications.

# Nice, but what does it all cost?

Jsonic is not going to be as fast at the builtin in JSON parser. Jsonic is for inline data in source code, settings, options, and configuration. Even if you have tens of thousands of line of configuration, Jsonic is plenty fast enough.

Fine, but does it scale? Yes, Jsonic is O(n). Larger and larger source text won't make the world implode.

Memory usage? It doesn't leak.

What about the call stack? All good. Jsonic doesn't use recursive functions (it's iterative) so can handle data of any depth, at any depth in the call stack of your system.

Every Jsonic release is measured for complexity, memory, and stack safety.

Bugs that break this commitment get the highest priority.

# Customizable and extendable

All of the JSON enhancements can be customized.

The top level Jsonic function cannot be customized. First, create a new instance with:

let myjsonic = Jsonic.make()

Then you can set some options. let's turn off comments:

myjsonic.options({comment:{lex:false}})

myjsonic('a:1,#b:2') // === {"a":1, "#b":2}
Jsonic('a:1,#b:2') // === {"a":1}

Maybe you're just against # style comments, but fine with //? OK, let's get rid of just # comments:

let nohash = Jsonic.make({comment:{marker:{'#':false}})

nohash('a:1,#b:2') // === {"a":1, "#b":2}
nohash('a:1,//b:2') // === {"a":1}

You can pass custom options directly to Jsonic.make when creating a new instance.`

All the options are explained in the options reference.

​Jsonic can be extended with plugins. A few are included in the standard package to get you started.

Let's parse some CSV:

const { Csv } = require('jsonic/plugin/csv')
let csv = Jsonic.use(Csv)

// === [{"a": 1, "b": 2}, {"a": 3, "b": 4}]
csv(`
a, b
1, 2
3, 4
`)

The available plugins are listed in the plugins section, along with tutorials to help you write your own plugins.
